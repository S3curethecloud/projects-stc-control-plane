Runtime Health

Use the Runtime page to confirm that the platform is healthy.

  • Status must be ok
  • Redis must be ok
  • Policy revision should match the current runtime environment
  • Active sessions shows total live sessions across tenants

Tenant Workflow

Typical tenant inspection workflow:

  1. Open Tenants page
  2. Select tenant
  3. Inspect usage metrics
  4. Inspect billing preview
  5. Inspect active sessions

Token Issuance Test

To simulate an agent request, issue a runtime token using the tenant API key. 

POST /v1/tokens/issue

{
  "principal": "agent:test",
  "intent": "refund:create",
  "scopes": ["refund:create"],
  "ttl_seconds": 600,
  "context": {
    "device_trust": true,
    "session_binding": "device-abc123",
    "risk_score": 10,
    "after_hours": false
  }
}

A valid request returns: 

status: issued
session_id
token (JWT)
expires_in

Session Revoke

Sessions can be revoked from the Sessions page or the Tenant Detail page.

  • Copy the session_id
  • Confirm revoke action
  • The runtime removes the session immediately

Revocation terminates the agent authorization immediately.
